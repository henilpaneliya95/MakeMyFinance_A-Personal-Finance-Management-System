MAKEMY FINANCE (MakeMy Finance) - COMPLETE WEBSITE WORKFLOW
============================================================

1) HIGH-LEVEL PROJECT FLOW
--------------------------
This project is divided into two major parts:

A) Frontend (React + Vite)
   Folder: frontend/
   Purpose: Render the UI, collect user input, and call backend APIs.

B) Backend (Django REST + MongoEngine)
   Folder: backend/
   Purpose: Authentication, business logic, CRUD operations, and ML/AI endpoints.

C) Data Storage
   - Primary data is stored in MongoDB Atlas (users, accounts, transactions, budgets, goals, and more).
   - SQLite is configured for Django defaults, but the app's domain and transactional data is handled through MongoEngine documents in MongoDB.


2) WHAT HAPPENS WHEN THE APP STARTS
------------------------------------
Frontend side:
- Entry file: frontend/src/main.jsx
- It renders the App component (frontend/src/App.jsx).
- Routes are defined in App.jsx using React Router:
  /            -> Home page
  /login       -> Login page
  /signup      -> Signup page
  /dashboard   -> Financial dashboard
  /transactions, /accounts, /analytics, /reviews, /budgets, /debt,
  /goals, /portfolio, /simulator, /help, /ai-insights, /scan

Backend side:
- Entry point: backend/main.py (runs Django manage commands)
- Main URL config: backend/item_service/urls.py
  - The /api/ route is mapped to backend/api/urls.py


3) FRONTEND -> BACKEND CONNECTION
---------------------------------
- API base URL: frontend/src/lib/api.js
  Default: http://localhost:8000/api

This means frontend data calls usually follow this pattern:
- ${API_BASE}/users/login/
- ${API_BASE}/transactions/
- ${API_BASE}/accounts/
- ${API_BASE}/ml/...


4) USER AUTH FLOW (SIGNUP + LOGIN)
----------------------------------
A) Signup:
- Page: frontend/src/pages/Signup.jsx
- The user enters username, email, and password.
- Frontend sends a POST request to: /api/users/signup/
- Backend SignupView creates the user.
- On success, the user is redirected to the login page.

B) Login:
- Page: frontend/src/pages/Login.jsx
- Frontend sends a POST request to: /api/users/login/
- Backend LoginView verifies email and password.
- Backend returns a JWT token.
- Frontend decodes the token and stores in localStorage:
  - accessToken
  - userId
  - username
  - email
- After login, frontend fetches user accounts and stores the first account as selectedAccountId in localStorage.
- Then it navigates to /dashboard.


5) ROLE OF SESSION / LOCAL STORAGE
----------------------------------
localStorage plays a central role in the frontend:
- accessToken: authentication state
- userId: mapping data to the current user
- selectedAccountId: currently active account

Many pages fetch data using these stored values.
Example:
- Dashboard loads data using accountId and userId.
- Transactions page shows data based on selectedAccountId.


6) DASHBOARD FLOW
-----------------
Page: frontend/src/pages/Dashboard.jsx

When Dashboard loads, these calls run:
1. /api/accounts/{selectedAccountId}/
   -> current account details
2. /api/transactions/accounts/{selectedAccountId}
   -> recent transactions
3. /api/ml/anomalies/users/{userId}/
   -> anomaly alerts

Then the frontend calculates income, expenses, balance, and savings rate, and shows them in cards/charts.

8) HOW AI / ML FEATURES WORK
----------------------------
Page: frontend/src/pages/Aiinsights.jsx

The AI Insights page calls ML endpoints based on userId:
- GET /api/ml/predict-expense/{userId}/
- POST /api/ml/anomalies/run/{userId}/
- GET /api/ml/anomalies/users/{userId}/
- POST /api/ml/recurring/run/{userId}/
- GET /api/ml/recurring/users/{userId}/
- GET /api/ml/goals/eta/{userId}/

Results are shown in the UI as forecasts, anomaly cards, recurring patterns, and goal ETA insights.


9) BACKEND ARCHITECTURE (SIMPLE VIEW)
-------------------------------------
Main backend components:
- URLs: backend/api/urls.py
- Views/business logic: backend/api/views.py
- Data models: backend/api/models.py
- Serializers/validation: backend/api/serializers.py

MongoEngine models include:
- User
- Account
- Transaction
- Budget
- Goal
- Portfolio
- Debt
- Review
- ContactMessage
- MLPrediction, SpendingAnomaly, RecurringPattern
- AIConversation, AISuggestion, FinancialInsight, UserPermission


10) SECURITY & AUTH NOTES
-------------------------
- Backend uses REST Framework with JWT authentication.
- After login, frontend stores the token and fetches user-specific data.
- For production, token storage and refresh flow can be hardened further (refresh endpoint is available at: /api/auth/refresh/).


11) END-TO-END EXAMPLE (REAL USER JOURNEY)
------------------------------------------
Step 1: User signs up -> account is created.
Step 2: User logs in -> JWT token is returned.
Step 3: Token is decoded and userId is stored in localStorage.
Step 4: A default account is selected (selectedAccountId).
Step 5: Dashboard fetches account + transactions + anomalies.
Step 6: User adds/edits/deletes transactions -> backend DB updates.
Step 7: Budget/Goals/Portfolio/Debt pages stay synced through their endpoints.
Step 8: AI Insights page calls ML APIs and displays predictive insights.


12) LOCAL RUN QUICK REFERENCE
-----------------------------
Backend:
- cd backend
- pip install -r requirements.txt
- python main.py runserver

Frontend:
- cd frontend
- npm install
- npm run dev

Default URLs:
- Frontend: http://localhost:5173 (or the Vite host port)
- Backend API: http://localhost:8000/api


13) IMPORTANT PRACTICAL NOTES
-----------------------------
- A duplicate project copy may exist in the workspace. While running commands, make sure you are in the correct active root.
- If no data appears after login, first check userId/selectedAccountId in localStorage.
- CORS settings are controlled in backend/item_service/settings.py via environment variables and DEBUG local origins.


This is the complete working overview: frontend UI -> API calls -> backend views/serializers -> MongoDB data -> response -> UI updates.
